Guia: Arch Linux Portátil (Minimal + Hyprland)

Este guia detalha como usar o archinstall para criar uma instalação mínima e portátil do Arch Linux em um pendrive ou SSD externo, focando em compatibilidade e longevidade, e como configurar o Hyprland manualmente após a instalação base.

Pontos-Chave para Portabilidade

  • Drivers Gráficos Universais: Instalação de todos os drivers open-source (mesa para Intel/AMD/Nouveau) para funcionar em diferentes máquinas.
  • Kernel Universal: Remoção do hook autodetect para que o kernel carregue todos os drivers de disco (SATA, NVMe, USB) durante o boot.
  • Swap Otimizado (zram): Uso de zram (swap comprimido na RAM) em vez de uma partição swap no disco, evitando desgaste.
  • Filesystem Otimizado (btrfs + noatime): Uso de btrfs com a opção noatime para reduzir drasticamente as escritas no pendrive/SSD.

O Que Você Vai Precisar

  • Pendrive Instalador: Um pendrive com a ISO mais recente do Arch Linux.
  • Pendrive/SSD Destino: Onde o Arch Linux será instalado (será formatado!).
  • Conexão com a Internet: Necessária durante a instalação.
  • Paciência e Atenção: Siga os passos cuidadosamente!

Guia de Instalação com archinstall

1
Inicialização e Internet

Dê boot no seu PC usando o Pendrive Instalador (ISO do Arch). Aguarde carregar até o prompt de comando (root@archiso ~ #).

Conecte-se à internet. O método mais comum é via Wi-Fi usando iwctl:

# Entra no utilitário iwctl
iwctl

# Lista suas placas de rede Wi-Fi (ex: wlan0)
[iwctl]# device list 

# Procura redes disponíveis (substitua wlan0 se necessário)
[iwctl]# station wlan0 scan

# Lista as redes encontradas
[iwctl]# station wlan0 get-networks

# Conecta à sua rede (substitua "NomeDaSuaRede")
[iwctl]# station wlan0 connect "NomeDaSuaRede" 
# (Será pedida a senha)

# Verifique se conectou (deve mostrar um IP)
[iwctl]# station wlan0 show

# Saia do iwctl
[iwctl]# exit

Teste a conexão: ping archlinux.org (Use Ctrl+C para parar).

2
Iniciando o archinstall

No prompt, simplesmente digite:

archinstall

Pressione Enter. O instalador baseado em menus será iniciado.

Navegação no Menu: Use as Setas Cima/Baixo para navegar, Enter para selecionar uma opção ou entrar em um sub-menu, e Espaço para marcar/desmarcar itens em listas de seleção múltipla (checklists).
3
Configurações de Idioma e Teclado
  • archinstall language: Navegue e selecione pt (Português).
  • Keyboard layout: Selecione br-abnt2 (ou o layout do seu teclado).
  • Mirror region: Selecione Brazil para acelerar os downloads.
  • Locale language: Selecione pt_BR.
  • Locale encoding: Selecione UTF-8.
4
Seleção e Formatação do Disco Destino
DICA: Verifique o Disco Correto! Antes de formatar, é crucial ter certeza de qual disco é o seu Pendrive/SSD Destino. Para isso:
  1. Pressione Ctrl+Alt+F2 para ir para outro terminal (TTY2).
  2. Digite lsblk e pressione Enter.
  3. Identifique seu pendrive destino pela capacidade (SIZE) e nome (NAME, ex: sdb, sdc). Anote o nome completo (ex: /dev/sdb).
  4. Pressione Ctrl+Alt+F1 para voltar ao instalador (TTY1).
  • Drive(s): Navegue com as setas e pressione Enter no seu Pendrive/SSD Destino (ex: /dev/sdb). Confirme que é o disco correto!
5
Layout do Disco e Filesystem (BTRFS)
  • Disk layout: Selecione a opção que diz algo como Wipe all selected drives and use a best-effort default partition layout (Limpar disco e usar layout padrão).
  • Filesystem: Selecione btrfs.
Por quê BTRFS? BTRFS é moderno, suporta compressão (economiza espaço e potencialmente reduz escritas) e snapshots (backups). É uma boa escolha para pendrives.
6
Swap (ZRAM)
IMPORTANTE: Não use swap em disco para pendrives!
  • Swap: Role para baixo na lista de opções (NÃO selecione apenas "True" ou "False"). Encontre e selecione a opção zram (makes a compressed swap in RAM).
Por quê ZRAM? ZRAM cria uma área de swap comprimida na sua memória RAM. Isso evita completamente as escritas de swap no seu pendrive/SSD, aumentando muito sua vida útil.
7
Opções de Montagem (noatime)
IMPORTANTE: Passo Crucial para Longevidade!

O instalador sugerirá subvolumes BTRFS (@, @home, @log, etc.) com opções padrão. Precisamos adicionar noatime a eles:

  1. Navegue até Mount options (ou similar) e entre.
  2. Você verá uma lista dos subvolumes (ex: / montado em @).
  3. Para CADA UM dos subvolumes principais (@, @home, @log, @var ou similares que apareçam):
    • Selecione o subvolume com Enter.
    • Você verá as opções atuais (ex: compress=zstd:1).
    • Edite a linha, adicionando ,noatime no final. O resultado deve ser, por exemplo: compress=zstd:1,noatime.
    • Pressione Enter para salvar a edição.
  4. Após editar todos, volte ao menu anterior.
Por quê noatime? Por padrão, o Linux atualiza o "tempo de acesso" de um arquivo toda vez que ele é lido. Isso causa muitas escritas pequenas e desnecessárias, especialmente ruins para pendrives/SSDs. noatime desativa essa atualização, reduzindo drasticamente o desgaste.
8
Bootloader
  • Bootloader: Selecione systemd-boot. É o padrão, simples e funciona bem para portabilidade UEFI.
9
Hostname e Contas
  • Hostname: Escolha um nome para a máquina (ex: arch-portatil).
  • Root password: Defina uma senha forte para o superusuário (root).
  • User account:
    • Selecione Add a user.
    • Digite um nome de usuário (ex: seu_nome).
    • Defina uma senha forte para seu usuário.
    • Quando perguntar Should seu_nome be a superuser (sudo)?, confirme com Yes (pressione Enter).
10
Perfil (Minimal)
IMPORTANTE: Selecione Minimal!
  • Profile:
    • Type: Selecione Minimal.
    • (Ignore Ambiente Gráfico/Placa Gráfica): O perfil Minimal não pedirá essas opções aqui. Vamos instalá-las manualmente depois.

Ao escolher "Minimal", garantimos que apenas o sistema base essencial será instalado, dando-nos controle total sobre o que adicionar depois.

11
Áudio
  • Audio: Selecione pipewire. É o servidor de áudio moderno e recomendado.
12
Kernel
  • Kernel: Selecione linux (o kernel padrão e estável).
13
Pacotes Adicionais
  • Additional packages: Deixe este campo em branco. Vamos instalar tudo o que precisamos manualmente após a instalação base.
14
Configuração de Rede
  • Network configuration: Selecione NetworkManager. É o gerenciador de rede mais comum e fácil de usar após a instalação.
15
Fuso Horário e NTP
  • Timezone: Selecione seu fuso horário (ex: America/Sao_Paulo ou America/Recife).
  • Automatic time sync (NTP): Selecione True.
16
Instalar!
  • Selecione a opção Install.
  • Revise todas as configurações mostradas na tela de confirmação.
  • Pressione Enter para iniciar a instalação.
  • Aguarde o processo terminar. Pode levar algum tempo dependendo da sua internet e do pendrive.

O "Truque" da Portabilidade (Passo Final)

17
Entrando no Novo Sistema (chroot)

Quando a instalação terminar, o archinstall fará uma pergunta crucial:

Would you like to chroot into the newly installed system to perform post-installation tasks?

  • Digite y (Sim) e pressione Enter.

Você agora está dentro do seu novo sistema Arch Linux no pendrive/SSD destino.

18
Removendo o autodetect

Este é o passo que garante a portabilidade. Precisamos editar o arquivo de configuração do kernel para que ele carregue *todos* os módulos, e não apenas os detectados na máquina atual.

Use o editor micro (que o archinstall geralmente inclui na ISO) ou nano para editar o arquivo:

# Recomendado: use micro se disponível
micro /etc/mkinitcpio.conf

# Alternativa: use nano se micro não estiver presente
# nano /etc/mkinitcpio.conf

Dentro do editor:

  1. Procure pela linha que começa com HOOKS=(...).
  2. Dentro dos parênteses, localize a palavra autodetect.
  3. Apague APENAS a palavra autodetect e o espaço antes ou depois dela. NÃO APAGUE MAIS NADA!
  4. Salve o arquivo e saia:
    • No micro: Ctrl+S para salvar, Ctrl+Q para sair.
    • No nano: Ctrl+O para salvar (Enter para confirmar), Ctrl+X para sair.
Alternativa com sed (Avançado): Se preferir, pode usar o comando sed para fazer a mesma coisa sem abrir o editor: 
sed -i 's/ autodetect//' /etc/mkinitcpio.conf
19
Regenerando o Kernel (Initramfs)

Agora, aplique a mudança que fizemos regenerando a imagem do kernel:

mkinitcpio -P

Aguarde o processo terminar. Você verá várias linhas sendo processadas.

20
Saindo e Reiniciando

Pronto! A instalação base e a configuração de portabilidade estão concluídas.

  1. Saia do ambiente chroot: 
    exit
  2. Reinicie o computador: 
    reboot
  3. MUITO IMPORTANTE: Assim que o PC começar a reiniciar, remova o Pendrive Instalador (ISO). Deixe apenas o seu novo Pendrive/SSD de sistema conectado.

Seu computador deve agora dar boot no seu novo Arch Linux portátil!

Pós-Instalação: Do Terminal ao Ambiente Gráfico

Após reiniciar, você será recebido por uma tela preta com um prompt de login (TTY). Aqui começa a configuração manual.

1
Login

Digite o nome de usuário que você criou (ex: seu_nome) e pressione Enter. Em seguida, digite sua senha (não aparecerá nada enquanto você digita) e pressione Enter.

Você estará no prompt de comando do seu usuário (ex: [seu_nome@arch-portatil ~]$).

2
Conectar à Internet (Novamente)

Como estamos no novo sistema, precisamos conectar à internet usando o NetworkManager que instalamos. Use o comando nmcli:

# Listar redes Wi-Fi disponíveis
nmcli device wifi list

# Conectar à sua rede (substitua "NomeDaSuaRede" e "SuaSenha")
nmcli device wifi connect "NomeDaSuaRede" password "SuaSenha"

# Verificar conexão (procure por "connected" e um IP)
nmcli connection show
ip address show

Teste com ping archlinux.org.

3
Atualizar o Sistema

É sempre bom começar atualizando todos os pacotes para as últimas versões:

sudo pacman -Syu

Digite sua senha de usuário quando solicitado. Confirme a atualização se houver pacotes para atualizar.

4
Criar Diretórios Padrão do Usuário

Para criar as pastas como ~/Documentos, ~/Imagens, etc., instale e rode o utilitário:

# Instala o pacote (pode já estar instalado como dependência)
sudo pacman -S --needed xdg-user-dirs 

# Cria/Atualiza as pastas
xdg-user-dirs-update
5
Instalando o Ambiente Gráfico (Hyprland + Essenciais)

Agora vamos instalar os pacotes necessários para o Hyprland e um ambiente funcional. Vamos dividir em grupos para entender melhor:

Grupo 1: Hyprland e Dependências Gráficas

sudo pacman -S hyprland xdg-desktop-portal-hyprland polkit-kde-agent qt5-wayland qt6-wayland
  • hyprland: O compositor Wayland em si.
  • xdg-desktop-portal-hyprland: Essencial para compatibilidade (compartilhamento de tela, seletores de arquivo).
  • polkit-kde-agent: Necessário para autenticação de privilégios (ex: GParted, configurações de sistema).
  • qt5-wayland, qt6-wayland: Camadas de compatibilidade para aplicativos Qt rodarem nativamente em Wayland.

Grupo 2: Terminal, Lançador e Ferramentas Básicas

sudo pacman -S kitty wofi micro firefox thunar waybar mako swaybg swaylock grim slurp wl-clipboard
  • kitty: Um terminal rápido e moderno, ótimo para Wayland.
  • wofi: Um lançador de aplicativos versátil (alternativa ao Rofi).
  • micro: Editor de texto simples no terminal (alternativa ao nano).
  • firefox: Navegador web.
  • thunar: Gerenciador de arquivos leve (GTK).
  • waybar: Barra de status/painel popular para Wayland.
  • mako: Servidor de notificações leve.
  • swaybg: Utilitário para definir papel de parede.
  • swaylock: Ferramenta para bloquear a tela.
  • grim: Ferramenta para tirar screenshots no Wayland.
  • slurp: Ferramenta para selecionar uma área da tela (usada com o grim).
  • wl-clipboard: Ferramentas para copiar/colar via linha de comando no Wayland.

Grupo 3: Fontes Essenciais e Ícones

sudo pacman -S noto-fonts noto-fonts-emoji ttf-font-awesome
  • noto-fonts, noto-fonts-emoji: Fontes padrão com boa cobertura de caracteres e emojis.
  • ttf-font-awesome: Fonte de ícones muito usada em barras como a Waybar.
Agrupando Comandos: Você pode instalar todos de uma vez se preferir, juntando os nomes dos pacotes em um único comando sudo pacman -S ....
6
Configuração Inicial do Hyprland

O Hyprland precisa de um arquivo de configuração para funcionar. Vamos criar um básico:

  1. Crie o diretório de configuração: 
    mkdir -p ~/.config/hypr
  2. Crie e edite o arquivo de configuração principal: 
    micro ~/.config/hypr/hyprland.conf
  3. Cole o seguinte conteúdo básico dentro do arquivo: 
    # Configuração Mínima Hyprland

# Monitor (ajuste se necessário, ex: eDP-1 para laptop)
monitor=,preferred,auto,1

# Programas de Inicialização
exec-once = waybar               # Inicia a barra Waybar
exec-once = swaybg -i ~/Imagens/wallpaper.png # Define um wallpaper (crie a pasta e coloque uma imagem!)
exec-once = /usr/lib/polkit-kde-authentication-agent-1 # Agente de autenticação

# Atalhos de Teclado Essenciais
\$mod = SUPER # Define a tecla "Windows" como modificador principal

bind = \$mod, RETURN, exec, kitty     # Abre o terminal Kitty com Super+Enter
bind = \$mod, Q, killactive,          # Fecha a janela ativa com Super+Q
bind = \$mod, M, exit,                # Sai do Hyprland (logout) com Super+M
bind = \$mod, E, exec, thunar         # Abre o Thunar com Super+E
bind = \$mod, D, exec, wofi --show drun # Abre o Wofi com Super+D
bind = \$mod, SPACE, togglefloating, # Alterna janela flutuante com Super+Espaço

# Mover foco com Super + Setas
bind = \$mod, left, movefocus, l
bind = \$mod, right, movefocus, r
bind = \$mod, up, movefocus, u
bind = \$mod, down, movefocus, d

# Mudar para Workspaces 1-5 com Super + Números
bind = \$mod, 1, workspace, 1
bind = \$mod, 2, workspace, 2
bind = \$mod, 3, workspace, 3
bind = \$mod, 4, workspace, 4
bind = \$mod, 5, workspace, 5

# Mover janela ativa para Workspaces 1-5 com Super + Shift + Números
bind = \$mod SHIFT, 1, movetoworkspace, 1
bind = \$mod SHIFT, 2, movetoworkspace, 2
bind = \$mod SHIFT, 3, movetoworkspace, 3
bind = \$mod SHIFT, 4, movetoworkspace, 4
bind = \$mod SHIFT, 5, movetoworkspace, 5

# Mover/Redimensionar janelas com mouse + Super
bindm = \$mod, mouse:272, movewindow
bindm = \$mod, mouse:273, resizewindow

# Configurações de Input (Teclado Brasileiro)
input {
    kb_layout = br
    kb_variant = abnt2
    follow_mouse = 1 # Foco segue o mouse
}

# Aparência Básica (Bordas, Gaps)
general {
    gaps_in = 5
    gaps_out = 10
    border_size = 2
    col.active_border = rgba(33ccffee) rgba(00ff99ee) 45deg
    col.inactive_border = rgba(595959aa)
    layout = dwindle # Layout padrão (espiral)
}

# Decoração (Bordas arredondadas, Sombra)
decoration {
    rounding = 10
    drop_shadow = yes
    shadow_range = 4
    shadow_render_power = 3
    col.shadow = rgba(1a1a1aee)

    # Configuração de Blur (Opcional, pode consumir mais recursos)
    # blur {
    #    enabled = true
    #    size = 3
    #    passes = 1
    # }
}

# Animações (Opcional)
# animations {
#    enabled = yes
#    bezier = myBezier, 0.05, 0.9, 0.1, 1.05
#    animation = windows, 1, 7, myBezier
#    animation = windowsOut, 1, 7, default, popin 80%
#    animation = border, 1, 10, default
#    animation = fade, 1, 7, default
#    animation = workspaces, 1, 6, default
# }

# Layout Dwindle (Espiral)
dwindle {
    pseudotile = yes 
    preserve_split = yes 
}
  4. Salve e saia (Ctrl+S, Ctrl+Q no micro).
  5. Crie a pasta de Imagens e coloque um arquivo wallpaper.png lá (ou ajuste o caminho no hyprland.conf): 
    mkdir ~/Imagens
# Agora, use um navegador ou outra ferramenta para baixar uma imagem
# e salvá-la como ~/Imagens/wallpaper.png
7
Iniciando o Hyprland (Pela Primeira Vez)

Agora, no terminal TTY, digite:

Hyprland

Pressione Enter. Seu ambiente gráfico Hyprland deve iniciar!

Explore os atalhos que definimos:

  • Super + Enter: Abrir terminal (Kitty)
  • Super + D: Abrir lançador (Wofi)
  • Super + Q: Fechar janela
  • Super + M: Sair do Hyprland (volta para o TTY)

Próximos Passos e Customização

Parabéns! Você tem um sistema Arch Linux portátil com Hyprland. Agora começa a diversão da customização.

Configurando sua Waybar

A configuração padrão da Waybar é muito básica. Para customizá-la:

  1. Crie os diretórios: mkdir -p ~/.config/waybar
  2. Copie os arquivos de exemplo: cp /etc/xdg/waybar/* ~/.config/waybar/
  3. Edite ~/.config/waybar/config (JSON) e ~/.config/waybar/style.css (CSS) ao seu gosto.
  4. Reinicie a Waybar (feche-a com killall waybar e reinicie via Wofi ou terminal) ou reinicie o Hyprland.
Ícones na Waybar: Certifique-se de ter a fonte ttf-font-awesome instalada e referenciada no seu style.css para os ícones aparecerem corretamente.
Instalando Mais Programas

Use o pacman para instalar mais aplicativos dos repositórios oficiais:

# Exemplo: Instalar GIMP (editor de imagem) e VLC (player de vídeo)
sudo pacman -S gimp vlc
Explorando o AUR (Arch User Repository)

Muitos programas (como Google Chrome, Spotify, temas específicos) não estão nos repositórios oficiais, mas sim no AUR. Para acessá-lo facilmente, instale um AUR helper como o yay:

# Instalar dependências para compilação
sudo pacman -S --needed base-devel git

# Clonar, compilar e instalar o yay
git clone https://aur.archlinux.org/yay.git
cd yay
makepkg -si

# Sair do diretório e limpá-lo (opcional)
cd ..
rm -rf yay

# Agora use 'yay' como o 'pacman' para instalar do AUR
# Exemplo: yay -S google-chrome
Cuidado com o AUR: Pacotes do AUR são mantidos pela comunidade. Verifique os PKGBUILDs antes de instalar.
Aprofundando a Configuração do Hyprland

Explore a Wiki oficial do Hyprland (https://wiki.hyprland.org/) para aprender sobre:

  • Binds Avançados: Criar atalhos mais complexos.
  • Window Rules: Definir comportamento específico para certas janelas (tamanho, posição, workspace inicial).
  • Variáveis de Ambiente: Forçar aplicativos a rodar em Wayland, definir temas, etc.
  • Monitores Múltiplos: Configurar diferentes resoluções e posições.
  • Animações: Customizar os efeitos visuais.

Edite seu ~/.config/hypr/hyprland.conf e adicione mais opções conforme aprende.

Referência: Programas Úteis e Comandos Essenciais

Comandos Essenciais do Pacman e Yay

  • sudo pacman -Syu: Sincroniza e atualiza todos os pacotes oficiais.
  • sudo pacman -S nome-do-pacote: Instala um pacote oficial.
  • sudo pacman -R nome-do-pacote: Remove um pacote oficial (mantém dependências).
  • sudo pacman -Rns nome-do-pacote: Remove um pacote oficial e suas dependências não usadas por outros.
  • pacman -Q nome-do-pacote: Verifica se um pacote oficial está instalado.
  • pacman -Qi nome-do-pacote: Mostra informações detalhadas sobre um pacote oficial instalado.
  • pacman -Ql nome-do-pacote: Lista os arquivos pertencentes a um pacote oficial instalado.
  • pacman -Qs termo-de-busca: Procura por pacotes oficiais instalados.
  • pacman -Ss termo-de-busca: Procura por pacotes oficiais nos repositórios.
  • yay -Syu: Sincroniza e atualiza pacotes oficiais E do AUR.
  • yay -S nome-do-pacote-aur: Instala um pacote do AUR (ou oficial).
  • yay -Rns nome-do-pacote: Remove um pacote (oficial ou AUR) e suas dependências.
  • yay -Qs termo-de-busca: Procura por pacotes instalados (oficiais ou AUR).
  • yay -Ss termo-de-busca: Procura por pacotes nos repositórios E no AUR.
  • yay -Scc: Limpa TODO o cache de compilação e download (libera espaço).
  • sudo pacman -Rns $(pacman -Qtdq): Remove pacotes órfãos (que não são mais dependências de nada).

Mais Dicas para Dominar o Hyprland

  • Organize sua Config: Use source= no hyprland.conf para separar binds, regras, startup, etc., em arquivos diferentes (ex: ~/.config/hypr/binds.conf). Isso facilita a manutenção.
  • Variáveis: Defina variáveis no início do seu hyprland.conf para reutilizar valores (ex: $terminal = kitty, depois use bind = $mod, RETURN, exec, $terminal).
  • Regras de Janela (Window Rules): Use hyprctl clients no terminal para descobrir a class e title de uma janela e criar regras específicas para ela (flutuar, mover para workspace, etc.) em windowrules.conf.
  • hyprctl: Aprenda a usar o comando hyprctl no terminal para interagir com o Hyprland (mudar configurações, obter informações, etc.) sem precisar editar o arquivo. Exemplos: hyprctl reload, hyprctl monitors, hyprctl dispatch killactive.
  • Logs: Se algo der errado, verifique os logs do Hyprland com journalctl --user -xe _COMM=Hyprland.
  • Recarregar Config: Após editar seus arquivos .conf, use o atalho Super + Shift + R (se definido) ou hyprctl reload no terminal para aplicar as mudanças sem reiniciar o Hyprland.
  • Backup (Dotfiles): Use git para versionar e fazer backup de seus arquivos de configuração (~/.config/hypr, ~/.config/kitty, etc.) em um repositório (ex: GitHub, GitLab). Isso é essencial!

Solução de Problemas Comuns

Pendrive Não Deu Boot?
  • Verifique se a BIOS/UEFI do computador está configurada para dar boot via USB e em modo UEFI (não Legacy/CSM).
  • Certifique-se de que o Bootloader (systemd-boot) foi instalado corretamente pelo archinstall.
  • Tente em outra porta USB ou em outro computador para isolar o problema.
Wi-Fi Não Funcionou Após Reiniciar?
  • Verifique o status do NetworkManager: systemctl status NetworkManager. Ele deve estar "active (running)".
  • Se não estiver ativo, tente iniciar: sudo systemctl start NetworkManager e habilitar para iniciar no boot: sudo systemctl enable NetworkManager.
  • Use nmcli device wifi list para ver se as redes aparecem. Se não, pode ser um problema de driver (verifique lspci -k).
Esqueci de Rodar mkinitcpio -P (ou removi autodetect errado)?
  • Dê boot novamente com o Pendrive Instalador (ISO).
  • Conecte-se à internet (iwctl).
  • Monte suas partições do Pendrive/SSD Destino. Exemplo (AJUSTE OS NOMES /dev/sdbX CONFORME SEU CASO! Use lsblk para verificar): 
    # Exemplo para BTRFS com subvolumes padrão do archinstall
mount /dev/sdb2 /mnt 
mount -o subvol=@home /dev/sdb2 /mnt/home
mount -o subvol=@log /dev/sdb2 /mnt/var/log 
# ... monte outros subvolumes se existirem (@var, etc)
mount /dev/sdb1 /mnt/boot # Partição EFI
  • Entre no sistema com arch-chroot /mnt.
  • Edite o /etc/mkinitcpio.conf (remova autodetect da linha HOOKS).
  • Execute mkinitcpio -P.
  • Saia com exit e reinicie com reboot (removendo a ISO).
